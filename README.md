# OptiMetrics\n\nOptiMetrics is a powerful Ad Performance Analytics platform tailored for Product Analysts and Ad Operations Teams. It processes ad metrics (CTR, CPC, Revenue, Impressions) and features an automated Insights Engine to detect anomalies, fatigue, and top-performing campaigns.\n\n## Features\n\n- **Comprehensive Dashboard:** Monitor total impressions, clicks, CTR, and revenue in real-time.\n- **Ad Rankings:** Deep-dive tabular view of all ads with advanced filtering and sorting.\n- **Segmentation Analysis:** Breakdown metrics by Device and Country.\n- **Insights & Alerts:** Rules-based engine highlighting ad fatigue, scaling opportunities, and device discrepancies.\n- **CSV Ingestion:** Bulk upload ad performance data effortlessly.\n\n## Tech Stack\n\n- **Framework:** Next.js (App Router)\n- **Language:** TypeScript\n- **Styling:** Tailwind CSS v4\n- **Charting:** Recharts\n- **Database:** PostgreSQL (via `pg`)\n\n## Getting Started\n\n### Prerequisites\n- Node.js >= 20.9.0\n- PostgreSQL Database\n\n### 1. Clone & Install\n```bash\n# Install dependencies\nnpm install\n```\n\n### 2. Environment Setup\nCreate a `.env.local` file at the root of the repository and add your Postgres credentials:\n```env\nDATABASE_URL=postgres://user:password@localhost:5432/optimetrics\n```\n\n### 3. Database Initialization\nRun the seeder script to initialize the schema and populate mock data (100,000 rows):\n```bash\nnode scripts/seed.mjs\n```\n\n### 4. Run the Dev Server\n```bash\nnpm run dev\n```\nOpen [http://localhost:3000](http://localhost:3000) to view the application.\n\n## Deployment\n\nThe dashboard is fully optimized for deployment on Vercel. Ensure you configure the `DATABASE_URL` environment variable in your Vercel project settings to point to your hosted PostgreSQL database (e.g. Supabase, Neon, or Vercel Postgres).\n
